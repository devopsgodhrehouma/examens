<h2>Projet — Application Web complète (Front Streamlit + Back Flask/FastAPI) + Docker</h2>

<p><strong>Objectif :</strong><br>
Réaliser une application complète en 2 parties :</p>
<ul>
  <li><strong>Front (obligatoire) :</strong> une interface <strong>Streamlit</strong> (UI).</li>
  <li><strong>Back (obligatoire) :</strong> une API <strong>FastAPI ou Flask</strong> (au choix de l’étudiant).</li>
  <li><strong>Déploiement (obligatoire) :</strong> conteneurisation avec <strong>Docker</strong> (idéalement <strong>Docker Compose</strong>).</li>
</ul>

<p><strong>Liberté de thème :</strong><br>
Le sujet est au choix (ex. gestion d’inventaire, mini-CRM, suivi d’habitudes, bibliothèque, suivi de dépenses, suivi d’entraînement, gestion de tickets, catalogue de produits, etc.).<br>
<strong>Important :</strong> le projet doit être inspiré du projet vu en classe (mêmes principes : UI séparée, API REST, échanges JSON, persistance, tests de base).</p>

<hr>

<h3>01 — Exigences minimales</h3>

<ul>
  <li><strong>Architecture 2 services :</strong>
    <ul>
      <li><strong>frontend</strong> : Streamlit</li>
      <li><strong>backend</strong> : Flask <em>ou</em> FastAPI</li>
    </ul>
  </li>
  <li><strong>API REST (obligatoire) :</strong>
    <ul>
      <li>au moins <strong>4 endpoints</strong> (ex. CRUD : Create/Read/Update/Delete)</li>
      <li>échanges en <strong>JSON</strong></li>
      <li>validation minimale des entrées</li>
    </ul>
  </li>
  <li><strong>Persistance (obligatoire) :</strong> SQLite (recommandé) ou fichier JSON/CSV (si très simple, moins de points)</li>
  <li><strong>UI Streamlit (obligatoire) :</strong>
    <ul>
      <li>au moins <strong>2 pages</strong> (ex. liste + détail / dashboard + formulaire)</li>
      <li>au moins <strong>1 formulaire</strong> (création ou mise à jour)</li>
      <li>affichage clair des résultats + messages d’erreur (validation)</li>
    </ul>
  </li>
  <li><strong>Docker (obligatoire) :</strong>
    <ul>
      <li>un <strong>Dockerfile</strong> pour le front</li>
      <li>un <strong>Dockerfile</strong> pour le back</li>
      <li>un <strong>docker-compose.yml</strong> pour lancer les deux services</li>
    </ul>
  </li>
</ul>

<hr>

<h3>02 — Livrables</h3>

<ul>
  <li>Un dépôt Git (ou un dossier compressé) contenant :
    <ul>
      <li><code>frontend/</code> (Streamlit)</li>
      <li><code>backend/</code> (Flask ou FastAPI)</li>
      <li><code>docker-compose.yml</code></li>
      <li><code>README.md</code> (installation, exécution, ports, endpoints, captures)</li>
    </ul>
  </li>
  <li><strong>Captures obligatoires</strong> :
    <ul>
      <li>Compose lancé (preuves : <code>docker compose ps</code> + logs)</li>
      <li>UI Streamlit fonctionnelle</li>
      <li>Au moins 2 appels API (ex. via navigateur, curl, Postman)</li>
    </ul>
  </li>
</ul>

<hr>

<h3>03 — Contraintes techniques (qualité minimale)</h3>

<ul>
  <li><strong>Ports :</strong> Streamlit sur <code>8501</code>, API sur <code>8000</code> (ou <code>5000</code> si Flask), clairement indiqués dans le README.</li>
  <li><strong>Connexion UI → API :</strong> l’UI appelle l’API via HTTP (ex. <code>requests</code>).</li>
  <li><strong>Variables d’environnement :</strong> l’URL du backend est configurable (ex. <code>BACKEND_URL</code>).</li>
  <li><strong>Gestion d’erreurs :</strong> si l’API est down ou renvoie une erreur, l’UI affiche un message clair.</li>
</ul>
