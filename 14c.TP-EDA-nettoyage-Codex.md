## 1. Contexte

Dans ce travail pratique, on dispose d’un fichier de données volumineux contenant des informations administratives sur des organismes, des bénéficiaires, des montants, des dates, des villes, des provinces et d’autres attributs descriptifs.

Ce fichier contient des problèmes fréquents dans les données réelles :

1. valeurs manquantes ;
2. doublons ;
3. colonnes inutiles ou peu informatives ;
4. formats de dates incohérents ;
5. textes écrits avec des variantes ;
6. noms d’organismes mal saisis ;
7. espaces inutiles ;
8. accents ou caractères spéciaux ;
9. valeurs qui semblent différentes mais qui représentent probablement la même entité.

Exemple :

```text
UQAM
U.Q.A.M.
AQAM
?QAM
Université du Québec à Montréal
Universite du Quebec a Montreal
UNIVERSITE DU QUEBEC A MONTREAL
```

Ces valeurs peuvent représenter la même organisation, mais elles ne doivent pas être corrigées automatiquement sans analyse.

Le fichier brut est disponible ici :

```text
https://drive.google.com/drive/folders/1sx-wsi0b25puWGidOJ6naSntieaHm-zv?usp=sharing
```

## 2. Objectif du travail

L’objectif est de préparer ce fichier pour une future utilisation dans un algorithme d’intelligence artificielle, notamment dans un contexte d’**apprentissage non supervisé**.

Avant de donner les données à un algorithme, il faut les nettoyer.

Un algorithme non supervisé peut regrouper des données, détecter des similarités ou trouver des structures cachées. Cependant, si les données sont sales, les résultats seront faux ou peu fiables.

Par exemple, si `UQAM`, `U.Q.A.M.`, `AQAM` et `?QAM` sont traités comme quatre organismes différents, l’algorithme peut créer de mauvais groupes.

Le but est donc de passer d’un fichier brut à un fichier propre, structuré et exploitable.

## 9. Étape 4 — Normalisation des textes

Créer le script suivant :

```text
src/04_detect_text_variants.py
```

Ce script doit analyser les colonnes textuelles importantes, en particulier les noms d’organismes.

Le script doit nettoyer les textes pour faciliter la comparaison.

Exemples de normalisation :

| Valeur originale                | Valeur normalisée               |
| ------------------------------- | ------------------------------- |
| U.Q.A.M.                        | UQAM                            |
| Université du Québec à Montréal | UNIVERSITE DU QUEBEC A MONTREAL |
| Universite du Quebec a Montreal | UNIVERSITE DU QUEBEC A MONTREAL |
| `?QAM`                          | QAM                             |
| `UQAM`                          | UQAM                            |

La normalisation doit inclure :

1. suppression des espaces au début et à la fin ;
2. mise en majuscules ;
3. suppression des accents ;
4. suppression des points inutiles ;
5. suppression des espaces multiples ;
6. nettoyage des caractères spéciaux.

## 14. Contraintes techniques

Le fichier peut être volumineux.

Le code doit donc être capable de fonctionner avec :

```text
chunksize
```

ou avec une stratégie équivalente.

Exemple :

```python
import pandas as pd

for chunk in pd.read_csv("data/raw/fichier_original.csv", chunksize=100000):
    print(chunk.shape)
```

Le projet doit être robuste et ne doit pas dépendre uniquement d’un petit échantillon.
