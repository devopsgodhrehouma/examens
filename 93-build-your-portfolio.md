# Exigences portfolio web

## 1. Pages obligatoires

Le portfolio doit contenir :

1. Accueil
2. À propos
3. Projets
4. Compétences
5. Blog
6. Contact
7. CV téléchargeable en PDF
8. Politique de confidentialité
9. Politique de cookies
10. Conditions d’utilisation
11. Mentions légales, si le site est utilisé dans un contexte professionnel ou commercial

## 2. Exigences SEO Google

Chaque page doit avoir :

1. Un titre unique
2. Une meta description
3. Une seule balise `H1`
4. Des URLs propres
5. Des images avec `alt`
6. Du contenu original
7. Aucun lien brisé
8. Aucune page importante bloquée par `robots.txt` ou `noindex`

## 3. Exigences blog

Le blog doit contenir au minimum :

1. Plusieurs articles originaux
2. Des titres clairs
3. Des dates de publication
4. Un auteur ou une signature
5. Des catégories ou mots-clés
6. Des liens internes entre les articles
7. Des articles structurés avec `H1`, `H2` et `H3`
8. Du texte réel, pas seulement des images ou des vidéos
9. Aucun contenu copié-collé sans source ou sans valeur ajoutée

## 4. Exigences AdSense et publicité

Pour pouvoir ajouter de la publicité avec Google AdSense, le site doit avoir :

1. Du contenu original et utile
2. Une navigation claire
3. Une page À propos crédible
4. Une page Contact visible
5. Une politique de confidentialité
6. Une politique de cookies
7. Aucune page vide ou en construction
8. Aucun clic artificiel sur les publicités
9. Aucun encouragement à cliquer sur les annonces
10. Aucun contenu trompeur, interdit ou de faible qualité

## 5. Performance

Le site doit être testé avec Lighthouse.

Scores minimaux :

| Critère        | Minimum |
| -------------- | ------: |
| Performance    |      85 |
| Accessibilité  |      90 |
| Best Practices |      90 |
| SEO            |      90 |

## 6. Qualité technique

Le projet doit inclure :

1. Code propre
2. Design responsive
3. Navigation claire
4. HTTPS si possible
5. Aucune clé API exposée
6. README complet sur GitHub

## 7. Projets à présenter

Minimum **3 projets**, avec pour chacun :

1. Description
2. Technologies utilisées
3. Captures d’écran
4. Lien GitHub
5. Lien démo si disponible

## 8. Livrables

À remettre :

1. Lien du site publié
2. Lien GitHub
3. Capture Lighthouse
4. CV PDF
5. README.md
6. Capture ou lien vers au moins un article de blog
7. Liens vers les pages légales

## 9. Barème

| Critère                |  Points |
| ---------------------- | ------: |
| Structure du portfolio |      15 |
| SEO                    |      15 |
| Projets                |      20 |
| Blog                   |      10 |
| Pages légales          |      10 |
| Performance            |      10 |
| Accessibilité          |      10 |
| GitHub / README        |       5 |
| Design                 |       5 |
| **Total**              | **100** |
