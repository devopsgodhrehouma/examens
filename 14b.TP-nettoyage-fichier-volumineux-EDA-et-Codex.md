# Exercice pratique — Nettoyage et analyse d’un fichier volumineux avec EDA et Codex

## 1. Contexte

Vous recevez un fichier de données volumineux contenant des informations administratives sur des organismes, des bénéficiaires, des villes, des provinces, des montants, des dates et plusieurs colonnes textuelles.

Le fichier contient des problèmes réels de qualité des données :

```text
UQAM
AQAM
?QAM
U Q A M
Université du Québec à Montréal
Université du Québec a Montreal
UNIVERSITE DU QUEBEC A MONTREAL
```

Ces valeurs peuvent représenter le même organisme, mais elles sont écrites de manière différente.

Le but de l’exercice est de préparer ce fichier pour une future analyse ou un futur modèle de Machine Learning.

# 8. Étape 1 — Inspection du fichier

## 8.1 Objectif

Analyser rapidement le fichier sans tout charger inutilement.

Le script `01_inspect_file.py` doit produire :

1. le nombre de colonnes ;

6. le nombre de lignes estimé ;
7. les colonnes numériques ;
8. les colonnes textuelles ;
9. les colonnes de date possibles.

---

## 8.2 Exemple de code attendu

```python
import pandas as pd
from pathlib import Path

DATA_PATH = Path("data/raw/fichier_original.csv")
REPORT_PATH = Path("reports/eda_report.md")

def inspect_file(path: Path) -> None:
    sample = pd.read_csv(path, nrows=1000)

    with open(REPORT_PATH, "w", encoding="utf-8") as report:
        report.write("# EDA Report\n\n")
        report.write("## 1. Aperçu général\n\n")
        report.write(f"Nombre de colonnes : {sample.shape[1]}\n\n")

        report.write("## 2. Colonnes\n\n")
        for col in sample.columns:
            report.write(f"- {col}\n")

        report.write("\n## 3. Types détectés\n\n")
        report.write(sample.dtypes.to_string())

        report.write("\n\n## 4. Aperçu des premières lignes\n\n")
        report.write(sample.head(10).to_markdown())

if __name__ == "__main__":
    inspect_file(DATA_PATH)
```

# 11. Étape 4 — Normalisation du texte

## 11.1 Objectif

Avant de comparer des noms, il faut les nettoyer.

Exemple :

```text
" Université du Québec à Montréal "
"UNIVERSITE DU QUEBEC A MONTREAL"
"Université-du-Québec-à-Montréal"
```

doivent être ramenés vers une forme comparable.

---

## 11.2 Fonction de normalisation attendue

Le script `04_detect_text_variants.py` doit contenir une fonction comme :

```python
import re
import unicodedata

def normalize_text(value: str) -> str:
    if value is None:
        return ""

    value = str(value).strip().upper()

    value = unicodedata.normalize("NFKD", value)
    value = "".join(
        char for char in value
        if not unicodedata.combining(char)
    )

    value = re.sub(r"[^A-Z0-9 ]", " ", value)
    value = re.sub(r"\s+", " ", value)

    return value.strip()
```
